Final Project Proposal

Ben Hoffman CTIS 210


For my final project, I'll make a brick breaker game (Option 1.) This will be a graphical 2D game where the player uses a paddle at the bottom of the screen (controlled by arrow keys) to bounce a ball up to break bricks at the top.


The game will include the following features:

  • A window the game is played in
  • A movable paddle at the bottom of the screen controlled by the left and right arrow keys
  • A ball that bounces off of bricks and the top and sides of the playing area, and can be bounced back up by the paddle.
  • The game is over when no bricks are left or 0 balls are above the paddle.
    • A corresponding win or loss screen is shown, with options for replayability. The win screen will have a place for the user to type their name to save it with their score
  • Leaderboard with high scores (low times)
  • Bricks break when the ball hits them
  • Keeping time and assigning a score based on how quickly the a level is completed
  • Textures for all entities in the game, as well as how the background, menus, leaderboard, etc will look

To construct this game, I will need to:

  • Create a Brick class to represent each brick
  • Create a Ball class to handle ball behavior
  • Create a Paddle class for paddle behavior
  • Store high scores and brick setup in their own files
  • Construct the graphical display of these elements, I think using Swing
  • Use keyboard / mouse listeners to handle paddle control
  • Track time and brick counting
  • Add necessary constructors, attributes, and methods to classes
  • Make ball physics
  • Check system’s time / date for recording with successful runs on the leaderboard


Research and Inspiration

I’ll probably need to research a range of topics including Java Swing, animation, physics, game development, and more. I also plan to look at open source brick breaker games for ideas and implementations if needed.
